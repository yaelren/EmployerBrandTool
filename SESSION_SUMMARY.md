## üéØ What We Built

### Problem Statement
The previous session implemented OAuth and dual save, but presets were still using data URLs (360KB for 2 images). Users wanted:
1. **Image picker** to browse pre-uploaded Wix Media Manager files
2. **Desktop upload** that creates permanent CDN URLs
3. **User choice** between Media Manager (permanent) and Local (temporary) uploads

### Solution: Backend API + Media Picker Modal

**Key Innovation:** CDN URLs are **240x smaller** than data URLs!
- Data URL preset: ~360KB
- CDN URL preset: ~1.5KB

## üèóÔ∏è Architecture

### Two-Server Approach

**Local Development:**
```
Backend API Server (Express)     Frontend Server (Live Server/Python/etc)
Port 3000                         Any port (5502, 8000, etc.)
‚îú‚îÄ /api/media/list               ‚îú‚îÄ index.html
‚îî‚îÄ /api/media/upload             ‚îú‚îÄ CSS files
                                 ‚îî‚îÄ JS files

**Production (Vercel):**
```
Vercel Deployment
‚îú‚îÄ Static Files (index.html, CSS, JS)
‚îî‚îÄ Serverless Functions
   ‚îú‚îÄ /api/media/list.js
   ‚îî‚îÄ /api/media/upload.js
```

### Why This Architecture?

**CORS Issue:**
```
Browser ‚Üí Wix Media Manager API
         ‚ùå BLOCKED by CORS (can't send custom headers)

Browser ‚Üí Backend API ‚Üí Wix Media Manager API
         ‚úÖ WORKS (backend adds headers)
```

**Auto-Detection:**
MediaPickerModal automatically detects environment:
- `localhost` ‚Üí Calls `http://localhost:3000/api`
- Production ‚Üí Calls `/api` (Vercel serverless functions)

    // Original upload logic for non-CDN images
    state.background.imageURL = await this.wixAPI.uploadMedia(...);
}
```

## üìä What Happens Behind the Scenes

### When You Browse Media Manager
```
[Browser]
   ‚Üí Click "üìÅ Browse Media Manager" button
   ‚Üí MediaPickerModal.show()
   ‚Üí Detects: localhost ‚Üí use http://localhost:3000/api
   ‚Üí fetch('http://localhost:3000/api/media/list')

[Express API Server - Port 3000]
   ‚Üí Receives GET /api/media/list
   ‚Üí Adds headers: Authorization, Content-Type, wix-site-id
   ‚Üí Calls Wix API: https://www.wixapis.com/site-media/v1/files
   ‚Üí Returns list of media files

[Browser]
   ‚Üí Receives JSON response
   ‚Üí Displays files in grid with thumbnails
   ‚Üí User clicks image
   ‚Üí Loads image from CDN URL
   ‚Üí Image displays in canvas
```

### When You Upload to Media Manager
```
[Browser]
   ‚Üí Click "‚ûï Upload New" in picker
   ‚Üí Choose file from desktop
   ‚Üí Upload mode dialog appears
   ‚Üí User selects "üìÅ Media Manager (recommended)"
   ‚Üí Creates FormData with file
   ‚Üí POST to http://localhost:3000/api/media/upload

[Express API Server]
   ‚Üí Parses multipart form data
   ‚Üí Reads file buffer

   Step 1: Generate Upload URL
   ‚Üí POST to Wix: /site-media/v1/files/generate-upload-url
   ‚Üí Receives: { uploadUrl, fileId }

   Step 2: Upload File
   ‚Üí PUT file buffer to uploadUrl

   Step 3: Get File Details
   ‚Üí GET from Wix: /site-media/v1/files/{fileId}
   ‚Üí Returns: { fileUrl, mimeType, width, height, ... }

   ‚Üí Sends response back to browser

[Browser]
   ‚Üí Receives CDN URL
   ‚Üí File appears in picker grid
   ‚Üí User can click to use it
   ‚Üí CDN URL ready for saving in preset
```

### When You Save Preset with CDN URLs
```
[PresetManager.savePreset()]
   ‚Üí Calls serializeState()
   ‚Üí For each image:

      Is this a CDN URL?
      ‚îú‚îÄ Yes (starts with https://static.wixstatic.com/media/)
      ‚îÇ  ‚Üí Save URL directly (tiny! ~50 bytes)
      ‚îÇ  ‚Üí Skip upload entirely
      ‚îÇ
      ‚îî‚îÄ No (data URL, object URL, etc.)
         ‚Üí Upload to Media Manager OR
         ‚Üí Fallback to data URL (~266KB)

[Result]
   Preset with 2 CDN URL images: ~1.5KB
   Preset with 2 data URL images: ~360KB

   **240x smaller!**
```

### When You Load Preset with CDN URLs
```
[PresetManager.loadPreset()]
   ‚Üí Calls deserializeState(presetData)
   ‚Üí For background image:

      imageURL exists?
      ‚îú‚îÄ Is CDN URL? ‚Üí Fetch from https://static.wixstatic.com/media/...
      ‚îî‚îÄ Is data URL? ‚Üí Decode base64 and create image

[Result]
   ‚Üí Image loads from Wix CDN (fast!)
   ‚Üí No upload needed
   ‚Üí Permanent, reliable URL
```

## üêõ Troubleshooting

### API Server Won't Start

**Error:** `Port 3000 already in use`

**Solution:**
```bash
lsof -ti:3000 | xargs kill
node dev-server.js
```

Or use different port (update MediaPickerModal.js):
```bash
PORT=3001 node dev-server.js
```

### "Failed to fetch media"

**Check:**
1. Is API server running? (Look for "üöÄ Media Manager API Server")
2. Is `.env` file correct?
3. Is API key valid?
4. Browser console for CORS errors?

**Test manually:**
```bash
curl http://localhost:3000/api/media/list
```

Should return JSON with files.

### "Upload failed"

**Check:**
1. File type supported? (images/videos only)
2. File size reasonable? (< 10MB recommended)
3. API key has Media Manager permissions?
4. Check API server terminal for error details

### Media Picker Shows Empty

**Possible causes:**
1. No files in Wix Media Manager yet
2. API key doesn't have read permissions
3. Site ID incorrect
4. API server not running

**Solution:** Upload first file manually in Wix Dashboard ‚Üí Media Manager ‚Üí Refresh picker
